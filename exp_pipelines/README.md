# 多流程设计方案对比实验（2026-06-12 晚）

目标：线上 61 分（旧基线被误交）→ 75+。在赛题合规边界内对比 4 种流程设计，
选出最优配方做全量重训和提交。

## 合规边界（所有方案共同遵守）

- 骨干仅用 OpenAI CLIP ViT-B/32 官方预训练权重（`timm/vit_base_patch32_clip_224.openai`）。
  方案 C 的高分辨率只是对官方权重的位置编码做双三次插值，权重本身不变，合规。
- 仅用本阶段官方训练数据；测试集只做推理。
- 单模型单推理流程（允许翻转 TTA），不做模型集成。
- 全流程脚本化可复现。

## 统一实验协议（保证可比性）

- 严格验证：先 90/10 分层划分（seed 42，与历史完全一致），kNN/教师/伪标签只用训练分区。
- 主指标：验证集 kNN 一致性中带（0.3–0.6）准确率 `mid_03_06`。
- 224 方案统一训练 12 轮；448 方案训练 8 轮（单轮约 3 倍耗时）。
- `--num-workers 2`（系统内存 15.7GB 承压，worker 数下调）。
- 每个方案独立 work-dir，互不覆盖，也不碰正式 `outputs/`。

## 方案总览

| 方案 | 目录 | 核心假设 | 关键改动（相对 A 基准） |
|---|---|---|---|
| A 基准（不重跑） | `outputs/`（引用今日下午记录） | 现行 v2 配方的水平 | attn-only LoRA r16 @224，ColorJitter，无 EMA |
| B 容量+正则 | `B_mlp_ema_aug/` | LoRA 容量是瓶颈，强增强+EMA 抑制噪声过拟合 | LoRA 扩到 MLP（1.27M→2.74M 参数）+ EMA 0.999 + RandAugment |
| C 高分辨率 | `C_hires448/` | ViT-B/32@224 仅 49 个 patch token，细粒度吃亏；448→196 token | B 配方 + `--img-size 448`（位置编码插值），8 轮 |
| D 数据召回 | `D_recall_more/` | 丢掉 25% 样本太浪费，连续权重足以压低噪声 | B 配方 + keep 0.85 + 伪标签阈值放宽（0.6/margin 0.05） |
| E 浅层适配 | `E_shallow6/` | 只适配后 6 块，保留 CLIP 底层通用特征，更抗噪 | B 配方 + `--lora-blocks 6` |

B/D/E 单轮约 5–6 分钟，C 单轮约 13–18 分钟；四个方案串行总计约 5 小时。

## 决策规则

1. 以 `mid_03_06` 最优者为胜出方案（A 基准参考值：12 轮 0.8762 / 15 轮 0.8780）。
2. 同时检查 `low_lt03` 不应显著上升（低带升高 = 拟合噪声的信号）。
3. 胜出方案用 `--full` 全量重训（如胜者是 C，可先 224 全量再 448 短调），
   `--predict --checkpoint full` 生成提交。
