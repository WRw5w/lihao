# 训练版本演进记录

> 噪声标签细粒度识别（CLIP ViT-B/32 鲁棒微调）的逐版本配方与结果变化。
> 记录截至 2026-06-15。所有版本骨干均为 CLIP ViT-B/32（赛规强制），单模型单推理。

## 0. 关键背景：指标与陷阱

- **线上真实指标**：干净、类别均衡测试集上的**整体 Top-1 Accuracy**（赛题第六节）。
- **本地验证指标**：90/10 分层划分（seed 42），两个代理：
  - `mid_03_06`：kNN 一致性 0.3–0.6 中带准确率（团队早期主指标）。
  - `noisy_all`：验证集整体准确率，**但量在脏标签上**。
- **核心教训**：⚠️ **两个本地指标都不可靠预测榜分**。`noisy_all` 在长训练后会因"记忆脏标签"而虚高，与干净榜分方向相反（见 v5）。**榜分是唯一真相。**

## 2. 推理端杠杆（不改训练，单模型合规）

针对真实指标（整体准确率）做的推理期增强：

| recipe | 做法 | 效果 |
|---|---|---|
| plain | 翻转 TTA | 基线 |
| **balanced** | logit 校正到均匀先验（测试集已知均衡） | 预测分布 std 19→1.4，改写~16%预测 |
| tta | 多尺度(448/512/576)+翻转 | — |
| **tta_balanced** | TTA + balanced | **🏆 榜单最大提分点** |
| gce / gce_balanced | GCE 鲁棒损失训练 + 上述推理 | 待榜分验证 |

**关键发现**：测试集均衡而原始预测严重偏斜（某类预测195次/应~50）→ **balanced 校正是合规免费的真杠杆**。

## 4. v5 实验：60 轮长调度（重要负结果）

**动机**：LR 是 `OneCycleLR(total_steps=epochs×steps)`，8 轮的 ep4 峰值疑为调度假象；长调度峰值或更晚更高。

**结果**：60 轮跑下，`noisy_all` 峰值后移到 **ep54=0.7941**（比 8 轮 0.7807 高 +1.34pt），top6 全在 ep41–60。**看似验证了假设。**

**但榜单打脸**：ep60 提交反而 **−1 分**。说明 `noisy_all` 的"提升"主要是**记忆脏验证标签**（过拟合噪声），不转化为干净准确率。

**定论**：✅ 调度确实决定峰位（ep4→ep8/54）；❌ 但长训练**不提升干净榜分**。**轮次不是杠杆，砍掉这条线。** 全量重训仍取 val 峰值轮（4–6轮）即可。

## 5. 当前最优组合 & 后续方向

**最优 = 结构 c448_dr_rank32_keep90（满数据、4轮）+ 推理 tta_balanced。**

- ❌ 已排除：512分辨率、长训练(60轮)、更高轮次。
- 🔍 待榜分验证：SWA 权重平均、GCE 鲁棒损失、balanced 强度扫描。
- 📌 纪律：往后所有实验**只认榜分**，noisy_all/mid 仅作参考。
- 🚫 不碰：多模型集成（违规且复赛/决赛复核必被取消）；如需"集成"收益，用 SWA 同模型权重平均（合规）。

## 附：环境与运维

- GPU RTX 5060 8GB / 系统 RAM 16GB（紧张）。448 全量单轮 ~10–13min，60轮 ~12h。
- DataLoader 锁页内存在低 RAM 下会僵死 → 全程 `--no-pin`。
- 复现：`finetune_lora.py`（训练）+ `tools/{tta_predict,balanced_predict,swa_soup}.py`（推理杠杆）。
