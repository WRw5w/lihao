# 冲分路线图：从 77.7 到 80+ / 85+

> 写于 2026-06-17，FET 首训进行中。核心问题：怎么用 FET 这条新线把分从汤的 77.7 拱到 80+ 甚至 85+。
> 方法论：**先诊断天花板卡在哪，再把杠杆对准最大头的瓶颈**。只认榜分，不臆断。

---

## 一、诊断：77.7 的天花板来自哪三个瓶颈

| 瓶颈 | 严重度 | 证据 | 能给的空间 |
|---|:--:|---|:--:|
| **A. 训练标签噪声(~37%错标)** | 🔴 最大 | clean_val 实验显示 kNN 多数票筛掉 37.1%；我们只保守丢 20%+回收84伪标签 | **最大(+3~8pt)** |
| **B. 冻结 CLIP B/32 表征上限** | 🟡 中 | 汤(同架构平均)封顶 77.7，扩池/多样化全失败→单架构到顶 | 中(+1~3pt) |
| **C. 空间分辨率(patch32 太粗)** | 🟡 中 | 448/32=196 token，每 patch 32px，亚-32px 细节被糊 | 中(+1~2pt) |
| D. 推理 | ⚪ 已榨干 | 满TTA+满均衡已 +3.4pt | 边际 |

**核心洞察**：
- **汤已经把瓶颈 B（架构）在"同一种全局特征"里榨到头了**——所以再多汤、再多样化都没用。
- **FET 攻的是 B**（加局部判别能力，正交维度）→ 但单靠架构大概只值 **+1~2pt（→~79）**。
- **真正的大头是 A（噪声）**。而 **FET 的价值不止是"更强分类器"，更是"更强的去噪老师"**：更强的模型 → 更干净的伪标签 → 重训更好 → 再去噪……**架构与去噪会复利叠加**。这才是冲 80+/85+ 的乘数。

> 一句话战略：**FET 架构(+arch) × 用 FET 当老师做激进去噪(+noise) 复利**，是唯一够空间到 85 的路；只调架构最多到 79~80。

---

## 二、分层方案（按"杠杆/瓶颈/预期增益/工作量"排序）

### TIER 1 — 让 FET 跑起来并调好（77.7 → ~79，低风险，本周）
攻瓶颈 B。
1. **首训 A/B**：当前 90/10 FET → 满推理 vs 77.73，看正交收益是否为正。
2. **旋钮扫**：`local_scale`(0.3/0.5/0.7)、`num_parts`(4/8/16)、`local_depth`(1/2/3)、`pfi_weight`(0/0.3/0.5 含消融)。
3. **un-detach 实验**：当前 `patch_feats.detach()` 让局部分支无法反向影响骨干。去掉 detach → 局部判别可经 LoRA 重塑骨干，可能是 +0.5 与 +2 的差别（小心显存/稳定）。
4. **FET 汤**：3~5 个 FET seed/config 同架构权重平均（合规，和 LoRA 汤一样）→ +1pt。

### TIER 2 — 分辨率与 token 丰度（并行，+1~2pt）
攻瓶颈 C。**FET 用空间 token，比只用 CLS 的 LoRA 更吃分辨率**（LoRA 在 512 反降，但 FET 可能受益）。
5. **FET 训/推 512 或 576**：更多 patch(16×16=256 / 18×18=324) → parts 分辨率更细。
6. **part-attention 引导的多裁剪 TTA**：FET 产出 `parts_mask`（判别区热力图）→ 裁判别区做 TTA，比盲目多尺度更准。

### TIER 3 — 直攻噪声（80 → 85 的胜负手，中高工作量）
攻瓶颈 A，**用 FET 当强老师**。这是空间最大的一层。
7. **迭代自蒸馏 / 重打标**：训好的 FET 对低可靠样本用"高置信预测"重打标 → 重训 → 重复 2~3 轮。每轮复利（更干净标签→更强模型→更干净标签）。便宜、直击主瓶颈。
8. **ELR（Early-Learning Regularization）**：一项正则把模型"早期学对、别记住噪声"锁住。**单 loss 项、改动极小、在 40~80% 噪声上被反复验证**——性价比最高，建议早接。
9. **Co-teaching**：两个模型（FET + LoRA）互相挑"小 loss 干净样本"喂对方，抗噪。**合规**：这是训练技巧，最终提交仍是单模型单推理。
10. **DivideMix（半监督去噪，SOTA）**：用每样本 loss 的 GMM 分干净/噪声；把噪声样本当**无标签**，加 MixMatch 一致性。40% 噪声的天花板方法，**空间最大但工作量最大**。

### TIER 4 — 全部叠加
最佳 FET 架构 + 最佳去噪(ELR/迭代/DivideMix) + 512 分辨率 + FET 汤 + 满均衡 TTA。

---

## 三、现实预期（诚实）

| 目标 | 可行性 | 路径 |
|---|:--:|---|
| 77.7 → **80** | **现实** | TIER1(FET +1~2) + ELR或迭代去噪(+1~2) + 512(+1) 任两项叠加 |
| 80 → **83** | 有机会 | TIER3 做扎实（迭代去噪/DivideMix 复利）+ FET 汤 |
| 83 → **85+** | 偏激进 | 需 DivideMix 级去噪做到位，且**可能撞冻结 B/32 天花板**——B/32 patch 粗 + 37% 噪声下，85 可能接近上限 |

> **85+ 的最大不确定性是瓶颈 B（冻结 B/32 上限）**。赛规锁死骨干，我们没法换 B/16。所以策略是把 A（噪声）和 C（分辨率）榨干，逼近 B/32 在干净数据上的天花板——那个天花板大概在 82~86 之间，能不能到 85 取决于它具体在哪。

---

## 四、建议执行顺序（最大性价比优先）

1. **现在**：FET 首训出分 A/B（确认架构正交收益为正）。
2. **立刻接 ELR**（改动最小、直击最大瓶颈）→ 这一步可能比调架构更值钱。
3. **迭代重打标**（用 FET+ELR 的强模型当老师，2~3 轮）→ 复利冲 80~82。
4. **并行**：512 分辨率 + FET 汤。
5. **若到 82 仍想冲 85**：上 DivideMix。
6. 全程：一次只改一处、各自和 77.73 单独 A/B、满推理、commit+自监督。

---

## 五、合规备注（重要）
- ELR / 迭代蒸馏 / Co-teaching / DivideMix **都是训练技巧**，最终提交仍是**单模型单推理** → 合规。
- FET 汤 / SWA = **同架构权重平均** → 合规（不是多模型推理集成）。
- 全程骨干冻结 + PEFT，满足赛规。
- ❌ 不碰：多模型推理融合/投票（违规，复赛复核必取消）。
